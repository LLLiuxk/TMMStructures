### 📅 [2026-03-29 更新] - 修复 Batch 2 分流种子 Bug

#### **A. Bug 根因定位**

在双机并行生成 Batch 2 时发现，`generate_batch2_part1.bat`（offset=0）和 `generate_batch2_part2.bat`（offset=25000）生成的样本内容**完全一致**，`--offset` 参数仅影响了文件命名而未影响随机数序列。

具体原因（`generate_dataset.py` → `build_dataset`）：

1. **全局种子初始化**：`random.seed(base_seed)` 直接使用 `config` 中的 `random_seed: 2026`，与 `offset` 无关。
2. **逐样本种子**：随机生成器中 `seed_val = base_seed + i`（`i` 从 0 开始循环计数），导致两台电脑的种子序列都是 `2026, 2027, 2028, ...`，输出完全相同。

#### **B. 修复方案（`generate_dataset.py`）**

两处修改，将 `offset` 纳入种子计算：

1. 全局种子：`random.seed(base_seed)` → `random.seed(base_seed + offset)`
2. 逐样本种子：`seed_val = base_seed + i` → `seed_val = base_seed + offset + i`

修复后效果：
- **Part 1** (offset=0)：种子序列 `2026, 2027, 2028, ...`（与已运行结果完全兼容，无需重跑）
- **Part 2** (offset=25000)：种子序列 `27026, 27027, 27028, ...`（产生完全不同的样本）

#### **C. 附加改进**

- 启动日志中新增 `Offset` 与 `sample ID 范围` 打印，方便确认分流配置正确。

### 📅 [2026-03-26] - 修复 C11 归零 Bug 与数据修正机制

#### **A. 零刚度 Bug 深度复盘**

在生成 `batch_1` 时遇到了所有力学刚度全部降为 $10^{-9}$ (Emin) 的严重 Bug。以下是问题根源完整溯源：

1. **生成期缺陷**：初始的图像均质化算法没有对 $128\times128$ 图像做4重周期性镜像对称，导致在施加周期性边界条件时，微结构边界断裂，无法传力。
2. **“假正常”的雷达图**：为什么属性为 $10^{-9}$，雷达图看起来仍有正常的形态？由于所有的属性值都坍塌到了惩罚下界，对数归一化的绘图脚本将其全部缩放至边界附近，形成了数值噪声伪影。
3. **修复与反转错误**：随后在试图修正该 Bug 时，引入了正确的 $256\times256$ 镜像机制，并在数据重新均质化期间错误地设置了 `invert=False`，导致生成的结构把实体变成了孔洞，属性依然为零。
4. **彻底解决**：统一使用 `invert=True` 读取黑底白图，同时加入 `OMP_NUM_THREADS=1` 等环境变量，彻底解决了 SciPy 求解器多核崩毁的问题。

#### **B. 评估指标与图表升级**

- 全面引入 **Bulk Modulus (体积模量 $K$)** 评价体系，取代单一的 $C_{11}$。
- 图表升级为学术级别的 **Log-Log 坐标 Ashby 点云图**，并且同步显示对应的 Hashin-Shtrikman 上界理论参考线，以准确表征结构效率。
- 将后台高耗时物理求解拆分为多进程（`recompute_dataset_mp.py`），加入 1000 样本自动持久化存档机制。

### 📅 [2026-03-20] - 修复均质化镜像 Bug 与对数雷达图重构

#### **A. 均质化几何镜像 Bug 彻底修复 (`homogenize.py`)**

- **Bug 根源**：之前版本中，均质化代码错误地将生成器输出的 `128x128` 四分之一晶胞 (Quarter Cell) 直接作为完整原胞计算，未执行周期性扩展，导致严重低估了部分正交方向的刚度（如 $C_{22} \\approx 1e-9$）。
- **优化方案**：重新在 `load_and_reconstruct` 函数中引入 4 倍镜像对称操作，正确将其补全为 `256x256` 完整晶胞再传入求解器。修正后极值结构的力热性能计算（$C_{11}/C_{22}$ 等比值）回归正常的物理区间。

#### **B. 强各向异性数据的可视化表达 (`plot_combined_radar.py`)**

- 传统的极坐标归一化对绝对性能差异达几个数量级的微结构无效，会在图上退化为“单线”或“点”。
- 现已替换为**对数拉伸归一化（Log-Scale Normalization）**，并保留微小的视觉非零底部基线，使得即使是极端结构也能拥有完整的形状轮廓，辅以刻度提示。

#### **C. 大规模并行重算套件 (`recompute_dataset_mp.py`)**

- 提供了利用 `multiprocessing` 加速的多核心重算脚本，以及配套的傻瓜式启动脚本 `recompute_dataset.bat`。
- 能够直接读取现有 `dataset_schema.json` 对原数据库执行物理场的清洗与重算覆盖，无需重头耗时生成新图像。
